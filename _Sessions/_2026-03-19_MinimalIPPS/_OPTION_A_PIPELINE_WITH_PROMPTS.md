# Option A: Pipeline Script + Prompt Pack

**Doc ID**: MIPPS-OPT-A
**Goal**: Reduce ~1MB DevSystem (rules, workflows, skills) to be usable by cheaper LLMs via automated compression pipeline
**Status**: Draft - pending user review

## Step-to-Output Mapping

Aligned with NOTES.md Steps 1-7:

- **Step 1**: Read all files → `context/all_files_bundle.md` (bundle command)
- **Step 2**: Analyze call tree → `_01_FILE_CALL_TREE.md` (Mother model)
- **Step 3**: Analyze complexity → `_02_FILE_COMPLEXITY_MAP.md` (Mother model)
- **Step 4**: Compression strategy → `_03_FILE_COMPRESSION_STRATEGY.md` (Mother model)
- **Step 5**: Generate prompts → `prompts/transform/` + `prompts/eval/` (Prompting model)
- **Step 6**: Transform files → `output/` (Transformer + Judge models, ensemble)
- **Step 7**: Verify results → `_04_FILE_COMPRESSION_REPORT.md` (Verification model)
- **Iterate**: Mother model reviews report, updates strategy → re-run Steps 6-7

## Pipeline Commands

```
mipps_pipeline.py bundle    --source-dir .windsurf/       → Step 1: generate context/all_files_bundle.md
mipps_pipeline.py analyze   --steps 2-4                   → Steps 2-4: Mother model, cached context
mipps_pipeline.py check     --step 2|3|4                  → Verify Mother output via Verification model
mipps_pipeline.py generate  --step 5                      → Step 5: Prompting model → transform/ + eval/
mipps_pipeline.py transform --step 6 --candidates 3       → Step 6: Transformer + Judge (ensemble)
mipps_pipeline.py verify    --step 7                      → Step 7: Verification model → report
mipps_pipeline.py iterate   --update-strategy             → Mother reviews report, updates _03_ strategy
mipps_pipeline.py status                                  → Show pipeline state, file counts, costs
```

## Model Roles

1. **Mother** - Claude Opus 4.6 Thinking (1M context)
   - Steps 2-4: Load ALL files as cached input, produce analysis documents
   - Iteration: Review compression report, update strategy
   - Ad-hoc: Answer verification questions from `check` command

2. **Prompting** - Claude Sonnet 4.5 Thinking (200K context)
   - Step 5: Generate transform/ and eval/ prompts
   - Input: compression strategy + file type samples (~50K tokens)

3. **Transformer** - GPT-5-mini (cost) or GPT-5.4 (quality)
   - Step 6: Compress files using transform/ prompts
   - Ensemble: N candidates per file, judge selects best, refine if below threshold
   - Same pattern as `transcribe-image-to-markdown.py`

4. **Verification** - GPT-5-mini or Claude Haiku 4.5
   - Step 6: Judge candidates using eval/ prompts
   - Step 7: Compare original vs compressed, produce per-file report
   - `check` command: Verify Mother outputs file-by-file, feed issues back to Mother

## Step 7 Report Format

Per NOTES.md, `_04_FILE_COMPRESSION_REPORT.md` contains 5 lines per file:

```
### rules/core-conventions.md
1. **Structural changes**: Merged 3 sections, flattened heading depth from 4 to 3
2. **Removed features**: Transcription Output rules, Temporary Files section
3. **Simplified content**: Date format examples reduced from 8 to 3
4. **Sacrificed details**: BAD/GOOD example pairs cut from 12 to 5
5. **Possible impact**: Agent may not apply datetime format in filenames correctly
```

## Verification Feedback Loop (Steps 2-4)

After Mother produces each analysis document, `check` command runs:

1. Verification model reads Mother output file-by-file
2. Compares claims against source files (spot-checks 10-20 files)
3. Reports issues (missing files, wrong counts, incorrect call chains)
4. Issues fed back to Mother (cached context) for correction
5. Corrected document saved

This prevents errors from propagating to the compression strategy.

## Cost Estimates

**Base assumptions**: ~1MB text = ~300K tokens, ~75 compressible files, 3 candidates

**IMPORTANT**: Claude Opus 4.6 Thinking bills thinking tokens as output tokens. Complex analysis over 300K context produces significant thinking overhead (~50-100K tokens per call).

**Steps 2-4: Mother Model (Claude Opus 4.6)**
- Pricing: $5.00/1M input, $0.50/1M cached, $25.00/1M output (includes thinking)
- Call 1 (call tree): 300K in × $5/1M + ~80K out × $25/1M = $1.50 + $2.00 = $3.50
- Call 2 (complexity, cached): 300K × $0.50/1M + ~90K out × $25/1M = $0.15 + $2.25 = $2.40
- Call 3 (strategy, cached): 300K × $0.50/1M + ~70K out × $25/1M = $0.15 + $1.75 = $1.90
- **Steps 2-4 total: ~$7.80**

**Steps 2-4 Verification (check command): GPT-5-mini**
- 3 documents × ~5 spot-check calls each = 15 calls × ~$0.01 = ~$0.15

**Step 5: Prompting Model (Claude Sonnet 4.5)**
- Pricing: $3.00/1M input, $15.00/1M output
- ~10 calls: 50K in + 10K out per call = $0.15 + $0.15 = $0.30/call
- **Step 5 total: ~$3.00**

**Step 6 Transform + Judge: GPT-5-mini**
- Transform: 75 files × 3 candidates = 225 calls × ~$0.01 = ~$2.25
- Judge: 225 calls × ~$0.004 = ~$0.90
- **Step 6 total: ~$3.15**

**Step 7 Verify: GPT-5-mini**
- 75 files × ~$0.005 = ~$0.38

**Iteration (Mother, cached): ~$2.50** (review report + update strategy)

**First iteration total: ~$16.50**
**Subsequent iterations: ~$8.30** (skip Steps 2-4, only re-run 6-7 + iterate)

**With GPT-5.4 as Transformer:**
- Step 6 transform: 225 × $0.08 = ~$18.00
- **First iteration with GPT-5.4: ~$32.30**

## Cost Optimization Levers

- **Batch API**: 50% discount on Steps 6-7 → saves ~$1.75
- **Reduce candidates**: 2 instead of 3 → saves ~$1.05
- **Selective re-runs**: Only re-transform failed files on iteration
- **GPT-5-nano** for judge/verify: $0.025/1M input → judge cost drops to ~$0.15
