# Container Files API

## Summary

Container Files API manages files within sandboxed containers. Upload files to a container (POST /v1/containers/{container_id}/files) by referencing file IDs from the Files API or uploading directly. Retrieve file metadata (GET), list files in a container (GET with pagination), delete files (DELETE), and download file content (GET content). Files uploaded to containers are accessible to Code Interpreter and Shell tools during execution. Files generated by code execution (charts, processed data, outputs) are also stored as container files and can be retrieved. Container files have a path property indicating their location in the container filesystem. Supported operations: create, retrieve, delete, list, and content download. Deleting a container removes all its files. [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)

## Key Facts

- **Upload methods**: Reference existing file_id or upload directly [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Generated files**: Code execution outputs stored as container files [VERIFIED] (OAIAPI-SC-OAI-GCODEI)
- **File path**: Files have a path in the container filesystem [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Content download**: Retrieve raw file content via GET content endpoint [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Cascade delete**: Deleting container removes all files [VERIFIED] (OAIAPI-SC-OAI-CNTAPI)

## Use Cases

- **Data upload**: Provide input files (CSV, JSON, images) for code execution
- **Output retrieval**: Download charts, reports, processed files generated by code
- **File management**: List and clean up files in long-lived containers
- **Multi-step workflows**: Upload data, process across multiple responses, download results

### Source Values

- **user**: Uploaded by the user
- **assistant**: Generated by code execution

## Operations

### Create (Upload File)

```
POST /v1/containers/{container_id}/files
```

**Method 1: Reference existing file**
```json
{
  "file_id": "file-abc123"
}
```

**Method 2: Upload with path**
```json
{
  "file_id": "file-abc123",
  "path": "/mnt/user/data.csv"
}
```

### List Files

```
GET /v1/containers/{container_id}/files
```

**Query Parameters:**
- **limit** (optional): 1-100, default 20
- **order** (optional): `asc` or `desc`
- **after** (optional): Cursor for pagination

### Retrieve File

```
GET /v1/containers/{container_id}/files/{file_id}
```

Returns file metadata including path, size, and source.

### Delete File

```
DELETE /v1/containers/{container_id}/files/{file_id}
```

### Download Content

```
GET /v1/containers/{container_id}/files/{file_id}/content
```

Returns the raw file content. Use for downloading generated charts, processed data, etc.

## SDK Examples (Python)

### Upload and Process Files

```python
from openai import OpenAI

client = OpenAI()

# Create container
container = client.containers.create(name="Analysis", memory_limit="4g")

# Upload file to container
with open("sales_data.csv", "rb") as f:
    file = client.files.create(file=f, purpose="assistants")

container_file = client.containers.files.create(
    container_id=container.id,
    file_id=file.id
)

print(f"Uploaded: {container_file.path}")

# Run analysis
response = client.responses.create(
    model="gpt-5.4",
    tools=[{"type": "code_interpreter", "container": container.id}],
    input="Read the CSV file and create a bar chart of sales by region. Save as chart.png"
)

print(response.output_text)
```

### Download Generated Files - Production Ready

```python
from openai import OpenAI
import os

client = OpenAI()

def download_container_files(container_id: str, output_dir: str, source: str = None):
    """Download files from a container"""
    os.makedirs(output_dir, exist_ok=True)
    downloaded = []
    after = None
    
    while True:
        params = {"container_id": container_id, "limit": 100}
        if after:
            params["after"] = after
        
        files = client.containers.files.list(**params)
        
        for f in files.data:
            if source and f.source != source:
                continue
            
            # Download content
            content = client.containers.files.content(
                container_id=container_id,
                file_id=f.id
            )
            
            # Save to output directory
            filename = os.path.basename(f.path) if f.path else f.id
            output_path = os.path.join(output_dir, filename)
            
            with open(output_path, "wb") as out:
                out.write(content.read())
            
            downloaded.append(output_path)
            print(f"Downloaded: {output_path} ({f.bytes} bytes)")
        
        if not files.has_more:
            break
        after = files.last_id
    
    return downloaded

try:
    # Download only assistant-generated files
    files = download_container_files(
        container_id="cntr_abc123",
        output_dir="./outputs",
        source="assistant"
    )
    print(f"Downloaded {len(files)} files")
except Exception as e:
    print(f"Error: {e}")
```

### Container Lifecycle Management

```python
from openai import OpenAI

client = OpenAI()

def cleanup_container(container_id: str):
    """List files, download outputs, then delete container"""
    # List all files
    files = client.containers.files.list(container_id=container_id, limit=100)
    
    print(f"Container {container_id}:")
    for f in files.data:
        print(f"  [{f.source}] {f.path} ({f.bytes} bytes)")
    
    # Delete container (removes all files)
    client.containers.delete(container_id)
    print(f"Deleted container {container_id}")

try:
    cleanup_container("cntr_abc123")
except Exception as e:
    print(f"Error: {e}")
```

## Error Responses

- **400 Bad Request** - Invalid file_id or path
- **404 Not Found** - Container or file not found
- **413 Payload Too Large** - File exceeds size limits

## Rate Limiting / Throttling

- **Upload**: Standard API rate limits
- **Content download**: Standard API rate limits
- **Storage**: Container storage counts toward organization limits

## Differences from Other APIs

- **vs Anthropic**: No container/sandboxed file management
- **vs Gemini**: Gemini Code Execution has implicit file handling, no explicit file management API
- **vs Grok**: No container file API

## Limitations and Known Issues

- **No directory listing**: Cannot list filesystem paths directly; use the files list endpoint [ASSUMED]
- **Binary content**: Content endpoint returns raw bytes; handle encoding appropriately [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Size limits**: Individual file upload size limits apply [ASSUMED]

## Gotchas and Quirks

- **Generated file discovery**: After code execution, list container files with `source=assistant` to find generated outputs [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Path matters**: Files are accessible at their container path during code execution [VERIFIED] (OAIAPI-SC-OAI-CNTFIL)
- **Cascade delete**: Deleting the container removes ALL files; download first if needed [VERIFIED] (OAIAPI-SC-OAI-CNTAPI)
